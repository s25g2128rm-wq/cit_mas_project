# Git 操作ガイド

このファイルはチームメンバー向けのGit基本操作マニュアルです。

---

## 1. 事前準備：Gitのインストール確認

コマンドプロンプト（またはPowerShell）を開いて以下を実行してください。

```bash
git --version
```

バージョン番号が表示されればOKです。
表示されない場合は [https://git-scm.com](https://git-scm.com) からインストールしてください。

---

## 2. リポジトリをクローンする（最初の1回だけ）

### 2-1. クローン先のフォルダを決める

クローンすると指定した場所にプロジェクトのフォルダが丸ごと作られます。
どこに置くかを先に決めておくと後で迷いません。

例：`C:\Users\あなたのユーザー名\Documents\projects` の中に置く場合

```bash
mkdir "C:\Users\あなたのユーザー名\Documents\projects"
cd "C:\Users\あなたのユーザー名\Documents\projects"
```

> `あなたのユーザー名` の部分は自分のPCのユーザー名に変えてください。
> エクスプローラーで `C:\Users\` を開くと自分のユーザー名が確認できます。

### 2-2. クローンする

```bash
git clone https://github.com/チームのユーザー名/リポジトリ名.git
```

クローンが完了すると、指定した場所に `リポジトリ名` というフォルダが作られます。

### 2-3. フォルダに移動する

```bash
cd リポジトリ名
```

以降の作業はすべてこのフォルダの中で行います。

---

## 3. 作業前に必ずやること：最新の状態に更新する

他のメンバーが変更を加えている可能性があるので、作業を始める前に必ずこれを実行します。

```bash
git pull
```

---

## 4. ファイルを編集したいとき

### 4-1. ブランチを作る

自分の作業用のブランチ（枝）を作ります。
mainブランチを直接編集しないのがルールです。

```bash
git checkout -b ブランチ名
```

ブランチ名の例：
- `plan-update-keisu`（計画案を更新するとき）
- `add-simulation-code`（コードを追加するとき）

### 4-2. ファイルを編集する

好きなエディタ（VSCodeなど）でファイルを編集してください。

### 4-3. 変更を保存する（コミット）

編集が終わったら変更を記録します。

```bash
git add .
git commit -m "変更内容をひとことで書く"
```

コミットメッセージの例：
- `"計画案Bを追加"`
- `"シミュレーションの初期実装"`
- `"READMEを修正"`

### 4-4. GitHubに送る（プッシュ）

```bash
git push origin ブランチ名
```

---

## 5. コードや計画案をmainに反映させたいとき（プルリクエスト）

1. GitHubのリポジトリページを開く
2. 「Compare & pull request」ボタンをクリック
3. 変更内容を説明する文を書いて「Create pull request」
4. チームメンバーに確認してもらってからマージ（mergeボタンを押す）

---

## 6. よく使うコマンド一覧

| やりたいこと | コマンド |
|---|---|
| 最新状態に更新 | `git pull` |
| ブランチを作る | `git checkout -b ブランチ名` |
| 今のブランチを確認 | `git branch` |
| 変更をステージに追加 | `git add .` |
| コミット | `git commit -m "メッセージ"` |
| プッシュ | `git push origin ブランチ名` |
| 変更状態を確認 | `git status` |

---

## 7. 困ったとき

- `git status` で今の状態を確認する
- 何かおかしくなったらkeisu（リポジトリオーナー）に声をかける
- 絶対にmainブランチを直接編集しない
