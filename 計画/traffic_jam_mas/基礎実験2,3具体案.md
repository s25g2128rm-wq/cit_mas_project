# 実験2・実験3：実施手順まとめ

「期待される結果」のグラフから逆算して、何をどう測れば確実にそのグラフが得られるかをまとめる。両実験とも構造は同じなので、まず共通の流れを示してから個別の詳細に入る。

---

## 0. 両実験に共通する基本構造

どちらの実験も、本質的には **「ある変数を動かして、各点で1回シミュレーションを走らせ、結果を集めてプロットする」** という同じパターンに従う。

```
for 変数 in 範囲:
    シミュレーションを走らせる
    過渡を除いた定常部分の統計を取る
    結果を保存
プロットする
```

このパターンに必要なものは以下：

| 必要なもの | 出どころ |
|---|---|
| シミュレーション本体 | `traffic_sim.py` の `run_simulation()` |
| 統計の集計 | `traffic_sim.py` の `summary_stats()` |
| ループ | 自分で書く（数行）|
| プロット | matplotlib |

つまり実装の本体は **10〜20行程度** で済む。

---

## 1. 実験2：N スイープ（臨界密度の発見）

### ゴールの再確認

最終的に作りたいグラフ：

- **横軸**：N（10〜40）または密度 ρ = N/350
- **縦軸**：定常状態の速度（m/s）
- **2本の線**：
  - 平均速度（青、丸マーカー）
  - 最低速度（赤、四角マーカー）
- **見えてほしい特徴**：ある N* を境に最低速度がゼロに張り付き始める

### 実施手順

#### Step 1：データ収集

```python
from traffic_sim import IDMParams, SimParams, run_simulation, summary_stats

n_values = list(range(10, 41))  # 10〜40を1刻み、31ケース
mean_v_list = []
min_v_list = []

for n in n_values:
    sim = SimParams(n_cars=n, n_steps=800)  # 過渡が抜けるよう余裕を持たせる
    result = run_simulation(sim=sim)
    stats = summary_stats(result, transient=300)  # 最初300ステップは捨てる
    mean_v_list.append(stats['mean_avg_v'])
    min_v_list.append(stats['mean_min_v'])
```

実行時間：31ケース × 50ms ≈ 2秒

#### Step 2：プロット

```python
import matplotlib.pyplot as plt

fig, ax = plt.subplots(figsize=(8, 5))
ax.plot(n_values, mean_v_list, 'o-', label='Mean velocity', color='C0')
ax.plot(n_values, min_v_list, 's-', label='Min velocity', color='C3')
ax.set_xlabel('Number of cars N')
ax.set_ylabel('Velocity (m/s)')
ax.set_title('Experiment 2: Velocity vs N')
ax.legend()
ax.grid(True, alpha=0.3)
plt.show()
```

#### Step 3：臨界密度の読み取り

グラフから「赤線（最低速度）がゼロに張り付き始める N」を目視で読み取る。N* / 350 で密度に換算。先行研究値（0.08〜0.10/m）と比較する。

### 注意点

- **過渡を十分に捨てる**：N が臨界付近だと波が立ち上がるまでに時間がかかる。`n_steps=800` で計算し、最初の 300 ステップを捨てるくらいの余裕が必要。
- **滑らかでない場合**：N=18〜25 あたりで線がガタつくことがある（リング上に立つ波の数が変わる相転移のため）。これ自体が興味深い現象なので、無理に滑らかにしようとしない。
- **N=10 など低密度では平均速度がデフォルトの 9.6 m/s 付近で安定するはず**（v₀ × 0.6 が初期速度のため）。もしこれより低ければバグの可能性。

## 2. 実験3：T スイープ（綱引きの実証）

### ゴールの再確認

最終的に作りたいグラフ：

- **横軸**：T（0.6〜2.0 秒）
- **縦軸**：定常状態の速度（m/s）
- **2本の線**：
  - 平均速度（青、丸マーカー）
  - 最低速度（赤、四角マーカー）
- **見えてほしい特徴**：T が小さいときは渋滞（最低速度ゼロ）、T が大きくなると渋滞が消えて速度が回復する

### 実施手順

#### Step 1：データ収集

実験2とほぼ同じ構造。違いは「動かす変数」だけ。

```python
import numpy as np
from traffic_sim import IDMParams, SimParams, run_simulation, summary_stats

t_values = np.arange(0.6, 2.05, 0.1)  # 0.6〜2.0を0.1刻み、15ケース
mean_v_list = []
min_v_list = []

for t in t_values:
    idm = IDMParams(T=t)
    sim = SimParams(n_cars=28, n_steps=800)  # N=28で固定（臨界付近）
    result = run_simulation(idm=idm, sim=sim)
    stats = summary_stats(result, transient=300)
    mean_v_list.append(stats['mean_avg_v'])
    min_v_list.append(stats['mean_min_v'])
```

実行時間：15ケース × 50ms ≈ 1秒

#### Step 2：プロット

```python
fig, ax = plt.subplots(figsize=(8, 5))
ax.plot(t_values, mean_v_list, 'o-', label='Mean velocity', color='C0')
ax.plot(t_values, min_v_list, 's-', label='Min velocity', color='C3')
ax.set_xlabel('Safe time headway T (s)')
ax.set_ylabel('Velocity (m/s)')
ax.set_title('Experiment 3: Velocity vs T')
ax.legend()
ax.grid(True, alpha=0.3)
plt.show()
```

#### Step 3：結果の解釈

- **左端（T=0.6）付近**：強い渋滞、最低速度がゼロに張り付く
- **右端（T=2.0）付近**：渋滞が消え、最低速度が平均速度と近づく
- **中間のどこか**：渋滞が消える境目がある（これが「T 軸での臨界点」）

### 注意点

- **N=28 で固定する理由**：実験2で確認した臨界密度付近に置く。ここで T を動かせば「綱引きの一方を緩めると渋滞が消える」が見える。N をもっと低くすると T を変えても何も起きない（既に安定領域なので）。
- **T が小さすぎると非物理的になる**：T < 0.5 程度になると車間が物理的に短すぎて IDM の挙動が破綻することがある。範囲は 0.6 以上にとどめるのが無難。
- **T を大きくすると平均速度が下がる現象**：T を増やしすぎると渋滞は消えるが、ドライバーが「臆病すぎて」遅くなる。最高速度になるのは「ちょうど波が消える T」のあたり。これも発表で言える面白い発見。

## 3. 2つの実験を1つのスクリプトにまとめる

実装上は、`run_sweep()` のような汎用関数を1つ書けば、N も T も同じ枠組みで処理できる。

```python
def run_sweep(varying_param, values, fixed_params=None):
    """
    1変数を動かしてスイープし、定常状態の統計を返す。
    """
    fixed_params = fixed_params or {}
    mean_v = []
    min_v = []
    for val in values:
        idm_kwargs = {k: v for k, v in fixed_params.items() if k in ['v0','T','a_max','b_comf']}
        sim_kwargs = {k: v for k, v in fixed_params.items() if k in ['n_cars','n_steps']}
        if varying_param in ['T', 'v0', 'a_max', 'b_comf']:
            idm_kwargs[varying_param] = val
        else:
            sim_kwargs[varying_param] = val
        result = run_simulation(IDMParams(**idm_kwargs), SimParams(**sim_kwargs))
        stats = summary_stats(result, transient=300)
        mean_v.append(stats['mean_avg_v'])
        min_v.append(stats['mean_min_v'])
    return mean_v, min_v


# 実験2：N スイープ
n_values = list(range(10, 41))
mean_v_n, min_v_n = run_sweep('n_cars', n_values, fixed_params={'n_steps': 800})

# 実験3：T スイープ
t_values = list(np.arange(0.6, 2.05, 0.1))
mean_v_t, min_v_t = run_sweep('T', t_values, fixed_params={'n_cars': 28, 'n_steps': 800})

# 2つを横に並べてプロット
fig, (ax1, ax2) = plt.subplots(1, 2, figsize=(14, 5))
# 実験2
ax1.plot(n_values, mean_v_n, 'o-', label='Mean', color='C0')
ax1.plot(n_values, min_v_n, 's-', label='Min', color='C3')
ax1.set_xlabel('N'); ax1.set_ylabel('Velocity (m/s)')
ax1.set_title('Exp2: N sweep'); ax1.legend(); ax1.grid(alpha=0.3)
# 実験3
ax2.plot(t_values, mean_v_t, 'o-', label='Mean', color='C0')
ax2.plot(t_values, min_v_t, 's-', label='Min', color='C3')
ax2.set_xlabel('T (s)'); ax2.set_ylabel('Velocity (m/s)')
ax2.set_title('Exp3: T sweep'); ax2.legend(); ax2.grid(alpha=0.3)
plt.tight_layout()
plt.savefig('experiments_2_and_3.png', dpi=120)
plt.show()
```

これで両方の結果が1枚の画像に収まる。

## 4. データの保存（試行錯誤を楽にするコツ）

プロットの色やラベル調整のために何度もシミュレーションを回すのは無駄。**まず numpy 配列として保存しておくと、後でプロットだけ調整できる**。

```python
import numpy as np

# 実験2のデータ保存
np.savez('exp2_data.npz', n=n_values, mean_v=mean_v_n, min_v=min_v_n)

# 後で読み込む
data = np.load('exp2_data.npz')
plt.plot(data['n'], data['mean_v'], ...)
```

特にプロット調整は試行錯誤が多いので、シミュレーション結果のキャッシュは効く。
